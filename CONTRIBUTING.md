# Contributing

PyDAVE is still in an experimental phase, but contributions and issue reports are welcome.

## Before changing code

Please read:
- `README.md`
- `docs/status.md`
- `docs/architecture.md`
- `docs/build/windows.md`

## Recommended contribution style

- keep low-level wrapper changes small and explicit
- prefer primary-source verification against upstream `libdave`
- add or update a smoke test when changing a native binding
- document notable build or ownership quirks when discovered

## Areas where feedback is especially valuable

- API ergonomics
- cross-platform build guidance
- test structure
- packaging and release design
- integration examples
